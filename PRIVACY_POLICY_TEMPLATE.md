## 4. How We Protect Your Data

### 4.1 Technical Security
- **Encryption in Transit:** All data transmitted over HTTPS/TLS encryption
- **Encryption at Rest:** Sensitive data encrypted in database
- **Password Hashing:** Passwords hashed using bcrypt (never stored as plain text)
- **Secure Token Storage:** Authentication tokens stored in encrypted device storage

### 4.2 Access Controls
- Role-based access control (RBAC)
- Employee access logging
- Regular security audits
- Principle of least privilege

### 4.3 Network Security
- gRPC with TLS for service communication
- Kafka with SASL/TLS for message broker
- Database connections over encrypted channels
- Database credentials never hardcoded in code

### 4.4 Data Validation
- Input validation on all endpoints
- SQL injection prevention (parameterized queries)
- Cross-site scripting (XSS) protection
- Rate limiting to prevent abuse

## 5. Data Retention

- **Account Data:** Retained while account is active. Deleted upon account deletion.
- **Transaction History:** Retained for 7 years for audit/compliance purposes
- **Device Data:** Deleted after 30 days of inactivity
